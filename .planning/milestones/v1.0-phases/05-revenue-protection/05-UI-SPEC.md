## Typography

Exactly 4 sizes and 2 weights are declared. No other sizes or weights may be introduced in this phase.

| Role | Size | Weight | Line Height | Font | Source |

Fonts are loaded via Google Fonts in `src/index.css`. No new font sizes introduced in this phase — all four sizes above are already present in the codebase.

## Component Inventory

New components introduced in this phase. All must follow patterns from `MarkFilledModal.tsx`.

### PlacementFeeModal

**Trigger:** Employer selects "Shortlisted" in the pipeline dropdown inside `ApplicantPanel`.

**Structure:**
- Backdrop: `fixed inset-0 z-40 bg-black/40`
- Container: `fixed inset-0 z-50 flex items-center justify-center p-4`
- Card: `bg-white rounded-[16px] shadow-xl w-full max-w-md border-[1.5px] border-fog`
- Header: icon (`Lock` from lucide-react, `w-5 h-5 text-hay`) + title "Shortlist Candidate" at 16px semibold + close `X` button (`aria-label="Close"`)
- Body sections (separated by `space-y-6`):
  1. Candidate name at 14px semibold text-ink
  2. Fee tier badge — pill showing tier label (`Entry`, `Experienced`, `Senior`) at 11px uppercase tracking-wide, `bg-hay-lt text-hay rounded-full px-2 py-1`
  3. Fee amount at 16px font-semibold text-ink (e.g. `$400 NZD`) — reuses modal heading size
  4. Explanatory text at 13px text-mid: "Placement fee applies if you hire this candidate. Shortlisting is free — this creates a record that you've seen their contact details."
  5. Contact details preview (blurred): phone and email placeholders at 13px font-mono text-mid blur-sm select-none
- Footer: two buttons full-width in `flex gap-3`
  - Cancel: `variant="outline"` (existing Button component) — label: "Keep current stage"
  - Primary CTA: `variant="primary"` (existing Button component) — label: "I understand — release contact details"

**Interaction contract:**
- Cancel: modal closes, pipeline dropdown resets to prior value, no DB write
- Confirm: Edge Function call writes `placement_fees.acknowledged_at`, then status updates to `shortlisted`, then modal closes, then ApplicantPanel re-fetches contacts (contacts appear unblurred inline)
- Loading state during confirm: button shows "Releasing..." with `disabled`, spinner icon from lucide-react (`Loader2` with `animate-spin`)
- Error state: `toast.error('Failed to release contact details. Please try again.')` via sonner — modal stays open

**Dropdown decoration (discoverable gate):**
- "Shortlisted" option in the transition Select component renders with `Lock` icon (16px, text-hay) prepended to the label
- Option label: "Shortlist — unlocks contact details"

### HireConfirmModal

**Trigger:** Employer selects "Hired" in the pipeline dropdown inside `ApplicantPanel`.

**Structure:**
- Same backdrop/container/card pattern as `PlacementFeeModal` and `MarkFilledModal`
- Header: icon (`CheckCircle` from lucide-react, `w-5 h-5 text-moss`) + title "Confirm Hire" at 16px semibold + close `X` button (`aria-label="Close"`)
- Body sections:
  1. Candidate label at 14px text-mid: "Confirm hire of [Seeker name/region]?"
  2. Invoice warning box: `rounded-[10px] p-4 bg-hay-lt border-[1.5px] border-hay` containing:
     - "A placement fee invoice of **$X NZD** will be generated and sent to your email." at 13px text-mid
     - "Payment due within 14 days." at 11px uppercase tracking-wide text-light (absorbed into 11px label size)
  3. Star rating (optional): five `Star` icons from lucide-react at 16px, unfilled `text-fog` / filled `text-hay`, click to rate 1-5. Label at 11px uppercase tracking-wide text-light: "Rate your matching experience (optional)"
- Footer: Cancel + "Confirm Hire" (variant="primary")
  - Cancel: `variant="outline"` — label: "Go back"
  - Primary CTA: `variant="primary"` — label: "Confirm Hire"
- Loading: "Confirming..." with Loader2 spin

**Interaction contract:**
- Cancel: modal closes, dropdown resets, no DB write
- Confirm: Edge Function call creates Stripe Invoice + writes `placement_fees.confirmed_at`, then status updates to `hired`, then modal closes
- Error: `toast.error('Failed to confirm hire. Please try again.')` — modal stays open
- Success: `toast.success('Hire confirmed — invoice sent to your email.')` — modal closes

### ApplicantPanel — Contact Section (new section)

Added between "Skills" section and "Pipeline Stage" section in the expanded panel.

**Condition for rendering:** `application.status` is `shortlisted`, `offered`, or `hired` (contacts section is not shown for earlier stages — nothing is blurred for pre-shortlist stages, contact section is simply absent).

**Structure:**
```
Section label: "Contact Details" — 11px font-body semibold uppercase tracking-wide text-light
Two-column grid (same pattern as Seeker Profile section):
  Phone label (11px text-light) / value (13px text-ink or blurred mono)
  Email label (11px text-light) / value (13px text-ink or blurred mono)
```

**Before acknowledgement (contacts prop null):** both values show blurred mono placeholders + a caption below the grid at 11px uppercase tracking-wide text-light italic: "Contact details unlock when you shortlist this candidate." (absorbed from 12px into 11px label size).

**After acknowledgement (contacts prop present):** real values, no blur. Phone shows "Not provided" in `text-light italic` if `contacts.phone === null`.

## Dropdown Decoration — Shortlisted Option

The existing `Select` component renders options from the `options` array. The transition options in `ApplicantPanel` must enrich the `shortlisted` option label to show the lock icon and gate description.

**Implementation note for executor:** Since the existing `Select` component accepts `{ value, label }` options, the label for `shortlisted` must be set to `"Shortlist — unlocks contact details"` (plain string). The `Lock` icon cannot be injected into a plain `<option>` element. If the Select uses Radix `SelectItem`, the icon can be rendered as a `ReactNode` — inspect `src/components/ui/Select.tsx` and use `SelectItem` with a leading icon if Radix is the underlying primitive. If native `<select>`, use the plain label string only.

## Interaction States

### PlacementFeeModal State Machine

```
idle → loading (on confirm button click)
  → success: write ack + status update → modal closes → contacts revealed inline
  → error: toast fires → modal stays open → idle
idle → cancelled (on cancel / backdrop click / X) → modal closes → dropdown resets
```

### HireConfirmModal State Machine

```
idle → loading (on confirm button click)
  → success: invoice created + status update → modal closes → toast success
  → error: toast fires → modal stays open → idle
idle → cancelled → modal closes → dropdown resets
```

### ApplicantPanel — Contact Reveal

```
contacts prop = null (not yet acknowledged):
  → render blurred placeholders
  → show "Contact details unlock..." caption

contacts prop = { phone, email } (acknowledged):
  → render real phone (or "Not provided" if null)
  → render real email
  → no blur, no caption
```

No animation is required on contact reveal — the panel simply re-renders with real data after the parent re-fetches post-acknowledgement. Consistent with existing pattern in ApplicantDashboard data flow.

## Email Template Design

Emails are sent via Resend. HTML is authored inline as template literal strings in the `send-followup-emails` Edge Function — no React Email framework for MVP (matches "Claude's Discretion" area in CONTEXT.md).

**Design constraints for email HTML:**
- Font stack: `DM Sans, -apple-system, Helvetica Neue, sans-serif` (DM Sans loaded via Google Fonts is unreliable in email clients — use system fallback)
- Background: `#F7F2E8` (cream) for email container, `#FFFFFF` white card content area
- Accent: `#2D5016` (moss) for primary CTA button background
- Text: `#1A1208` (ink) for body, `#6B5D4A` (mid) for secondary text
- Max-width: 560px centered
- No external image assets (logo as text: "TopFarms" in `#2D5016` at 16px)
- CTA button: `background:#2D5016; color:#ffffff; border-radius:8px; padding:16px 24px; text-decoration:none; font-weight:600`
