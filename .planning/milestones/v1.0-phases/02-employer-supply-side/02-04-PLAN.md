**PostJob.tsx — Job Posting Wizard Shell**
Per user decision: "Auto-save drafts — job starts as draft on first screen, auto-saves progress, employer can return to drafts from dashboard"

Pattern similar to EmployerOnboarding shell but with job-specific logic:

```typescript
// Core flow:
// 1. Check URL params for ?job_id= (editing existing draft)
// 2. If job_id param exists: load draft from jobs table, set currentStep from saved progress
// 3. If no job_id: start fresh wizard
// 4. Step 1 submission: INSERT new job row with status='draft', get job_id back
// 5. Steps 2-5: UPDATE existing job row by job_id
// 6. Step 6 (Preview): read-only summary, no DB write
// 7. After preview approval: navigate to payment step (Plan 05 handles this)
```

- Wrap in DashboardLayout
- Load employer_profiles to get employer_id (needed for job INSERT)
- Guard: if employer.onboarding_complete is false, redirect to /onboarding/employer with toast "Complete your farm profile first"
- StepIndicator with labels: ['Basics', 'Farm Details', 'Skills', 'Compensation', 'Description', 'Preview', 'Payment']
  - Note: 7 total steps including payment, but this plan builds steps 0-5 (6 content steps). Step 6 (payment) is built in Plan 05.
- Track `jobId` in state; set after step 1 insert
- Store the sector from step 1 in state to pass to SkillsPicker in step 3

```typescript
z.object({
  salary_min: z.coerce.number().min(0, 'Enter minimum salary').optional(),
  salary_max: z.coerce.number().min(0).optional(),
  benefits: z.array(z.string()),
}).refine(
  (data) => !data.salary_min || !data.salary_max || data.salary_min <= data.salary_max,
```
