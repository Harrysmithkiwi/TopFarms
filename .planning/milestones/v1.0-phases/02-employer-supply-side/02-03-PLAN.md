**EmployerVerification.tsx — Verification Hub**
This is the main verification page, accessible from employer dashboard.

- Wrap in DashboardLayout
- Load employer profile ID: `supabase.from('employer_profiles').select('id').eq('user_id', user.id).single()`
- Use `useVerifications(employerProfileId)` hook to load current verification state
- Show VerificationBadge at top with current trustLevel
- Show 5 verification cards in a grid (2 columns on desktop, 1 on mobile):

1. **Email** — Always "Verified" (green check). Auto-verified at signup. No action needed.
   - Create email verification record on mount if it doesn't exist: upsert to `employer_verifications` with `{ employer_id, method: 'email', status: 'verified', verified_at: new Date() }`

2. **Phone** — Shows status. CTA: "Verify Phone" → expands inline to show PhoneVerification component.

3. **NZBN** — Shows status. CTA: "Submit NZBN" → expands inline to show NzbnVerification component.
   Per user decision: "NZBN verification shows nothing externally until admin confirms — employer sees 'Pending Review' in their own settings only"

4. **Documents** — Shows status. CTA: "Upload Documents" → links to `/dashboard/employer/verification/documents`

5. **Farm Photos** — Shows status. CTA: "Upload Photos" → links to `/dashboard/employer/verification/photos`

Each card shows: icon, method name, status badge (Verified/Pending/Not Started), action button.

**PhoneVerification.tsx**
Inline component rendered within the verification hub card.

Two states:
1. **Enter phone**: Input for phone number (NZ format +64...), "Send Code" button
   - On submit: `await supabase.auth.updateUser({ phone: phoneNumber })`
   - Handle errors (show inline error message)
   - Success: switch to OTP entry state

2. **Enter OTP**: 6-digit code input, "Verify" button
   - On submit: `await supabase.auth.verifyOtp({ phone, token: code, type: 'phone_change' })`
   - On success: upsert employer_verifications: `{ employer_id, method: 'phone', status: 'verified', verified_at: now() }`
   - Call `refresh()` from useVerifications
   - Show success toast

- "Resend Code" link with 60-second cooldown timer
- Note: Phone OTP requires SMS provider configured in Supabase. If `updateUser` fails, show helpful error: "Phone verification is not yet configured. Please contact support."

**NzbnVerification.tsx**
Per user decision: "Manual NZBN verification (admin flag, no API integration for MVP)"

Inline component:
- Input field for NZBN number (13-digit New Zealand Business Number)
- Basic validation: must be numeric, 13 digits
- On submit: upsert employer_verifications: `{ employer_id, method: 'nzbn', status: 'pending', nzbn_number: value }`
- Show "Pending Review" status with clock icon after submission
- Cannot re-submit once status is 'pending' — show read-only view with the submitted number
- If admin has set status to 'verified': show green check
- If admin has set status to 'rejected': show red X with "Please resubmit"

**DocumentUpload.tsx** (EVER-04)
Per user decision: "Drag-and-drop upload zones with thumbnail preview for documents and farm photos — accept common image/PDF formats with clear file size limit"

- Wrap in DashboardLayout
- Page title: "Upload Verification Documents"
- Description: "Upload business registration, farm ownership, or other verification documents"
- Use FileDropzone component with:
  - `bucket="employer-documents"`
  - `path="${userId}/documents"`
  - `accept={{ 'image/*': ['.png', '.jpg', '.jpeg'], 'application/pdf': ['.pdf'] }}`
  - `maxSize={10 * 1024 * 1024}` (10MB)
  - `label="Drag and drop your document, or click to browse"`
- On upload complete: upsert employer_verifications: `{ employer_id, method: 'document', status: 'verified', document_url: url, verified_at: now() }`
- Show list of previously uploaded documents (query employer_verifications where method='document')
- Call `refresh()` on useVerifications after upload
- Back button → /dashboard/employer/verification
- Show success toast on upload

**FarmPhotoUpload.tsx** (EVER-04)
Similar to DocumentUpload but for farm photos:

- Page title: "Upload Farm Photos"
- Description: "Show seekers what your farm looks like — photos help build trust"
- Use FileDropzone with:
  - `bucket="employer-photos"` (public bucket)
  - `path="${userId}/farm"`
  - `accept={{ 'image/*': ['.png', '.jpg', '.jpeg', '.webp'] }}`
  - `maxSize={10 * 1024 * 1024}` (10MB)
  - `label="Drag and drop farm photos, or click to browse"`
- On upload complete: upsert employer_verifications: `{ employer_id, method: 'farm_photo', status: 'verified', document_url: url, verified_at: now() }`
- Show grid of previously uploaded photos with thumbnails
- Allow multiple uploads (each new upload updates the verification record URL to latest, but display all uploaded photos)
- For multiple photos: store URLs in a separate query or append to a photos array. Simplest approach: store latest photo URL in verification record, but render all photos from storage listing (`supabase.storage.from('employer-photos').list(\`${userId}/farm\`)`)
