**EmployerOnboarding.tsx — Wizard Shell**
Per user decision: "Auto-save progress after each screen completes — employer returns to where they left off" and "Strict linear order through all 8 screens (no skipping)".

Pattern: Parent component owns step state. Uses `useWizard` hook for navigation. Loads existing profile on mount to resume progress.

```typescript
// Core logic:
// 1. On mount: query employer_profiles by user_id to get onboarding_step
// 2. If profile exists: setInitialStep(data.onboarding_step), load existing data
// 3. Each step renders its own RHF form with Zod schema
// 4. On step submit: call handleStepComplete(stepData, stepNumber)
// 5. handleStepComplete: upsert to employer_profiles with {...stepData, onboarding_step: step + 1}
// 6. If step === 7 (last): also set onboarding_complete: true
```

- Wrap in DashboardLayout
- Render StepIndicator at top with labels: ['Farm Type', 'Farm Details', 'Culture', 'Accommodation', 'Verification', 'Pricing', 'Preview', 'Complete']
- Conditionally render the correct Step component based on currentStep
- Pass `onComplete: (data) => handleStepComplete(data, stepIndex)` and `defaultValues` (loaded from existing profile) to each step
- Pass `onBack: () => prevStep()` to steps 1-7 (not step 0)
- Loading state while fetching profile on mount (show skeleton/spinner)
