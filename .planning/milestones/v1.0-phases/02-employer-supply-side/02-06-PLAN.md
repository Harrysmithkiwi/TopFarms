**pg_cron migration — 008_job_expiry_cron.sql**
Per research: "pg_cron scheduled daily job updating status='expired' where expires_at < now() and status='active'"

```sql
-- Schedule daily expiry check at 2:00 AM UTC
SELECT cron.schedule(
  'expire-job-listings',
  '0 2 * * *',
  $$
    UPDATE public.jobs
    SET status = 'expired'
    WHERE status = 'active'
      AND expires_at < now();
  $$
);
```

Note: pg_cron must be enabled in Supabase project settings. If not available, add a comment noting this requires the pg_cron extension. Check STATE.md blocker: "Verify pg_cron vs Supabase scheduled functions availability".
