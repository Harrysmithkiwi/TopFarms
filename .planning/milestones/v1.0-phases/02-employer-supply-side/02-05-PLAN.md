**create-payment-intent/index.ts**

Deno Edge Function that creates a Stripe PaymentIntent or handles free first listing.

Per user decision: "First listing free: show all three tiers with $0 price and 'First listing free!' badge — employer still picks a tier to understand future pricing"
Per research pitfall: "Check listing_fees count in the Edge Function before creating the PaymentIntent" to prevent race condition.

```typescript
// Request body: { job_id: string, tier: 1 | 2 | 3, employer_id: string }
// Response: { client_secret: string | null, is_free: boolean }
```

Logic:
1. Validate request body (job_id, tier, employer_id required)
2. Create Supabase service role client
3. Verify the job belongs to the employer: `supabase.from('jobs').select('id, employer_id').eq('id', job_id).eq('employer_id', employer_id).single()`
4. Check listing_fees count for this employer: `supabase.from('listing_fees').select('id', { count: 'exact' }).eq('employer_id', employer_id)`
5. If count === 0 (first listing free):
   - Insert listing_fee with amount_nzd=0, tier, paid_at=now(), stripe_payment_id=null
   - Update job: status='active', listing_tier=tier, expires_at=now()+30 days
   - Return `{ client_secret: null, is_free: true }`
6. If count > 0 (paid listing):
   - Create Stripe PaymentIntent: amount from TIER_PRICES, currency='nzd', metadata={job_id, tier, employer_id}
   - Return `{ client_secret: paymentIntent.client_secret, is_free: false }`

TIER_PRICES: `{ 1: 10000, 2: 15000, 3: 20000 }` (cents NZD)

Import Stripe from `https://esm.sh/stripe@14`. API version: `'2024-06-20'`.

Add CORS headers for browser access:
```typescript
const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
}
```

Handle OPTIONS preflight request.

**stripe-webhook/index.ts**

Deno Edge Function handling Stripe webhook events.

Follow the exact pattern from 02-RESEARCH.md "Stripe Webhook Edge Function" code example:

1. Verify webhook signature using `stripe.webhooks.constructEvent(body, signature, STRIPE_WEBHOOK_SECRET)`
2. Handle `payment_intent.succeeded`:
   - Extract `job_id`, `tier`, `employer_id` from `pi.metadata`
   - Idempotency check: query `listing_fees` by `stripe_payment_id = pi.id`. If exists, skip.
   - If not exists:
     - Insert listing_fee: `{ job_id, employer_id, tier, amount_nzd: pi.amount, stripe_payment_id: pi.id, paid_at: now() }`
     - Update job: `{ status: 'active', listing_tier: tier, expires_at: 30 days from now }`
3. Return 200 for all events (including unhandled ones)

Per research pitfall: "Use INSERT ... ON CONFLICT DO NOTHING" is not directly available via Supabase JS, so use the check-then-insert pattern shown above for idempotency.

Do NOT authenticate the request via Supabase auth (webhook comes from Stripe, not a user). Use service role key.
