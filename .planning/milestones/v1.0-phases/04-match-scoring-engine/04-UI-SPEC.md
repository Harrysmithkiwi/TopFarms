## Spacing Scale

Tailwind v4 default scale (multiples of 4). The project uses inline Tailwind utility classes — no custom spacing tokens are declared in `@theme`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (p-1, gap-1) | Icon gaps, tight inline padding |
| sm | 8px (p-2, gap-2) | Compact element spacing, low-context hints below dimension rows |
| md | 16px (p-4, gap-4) | Default element spacing, card internal padding baseline |
| lg | 24px (p-6, gap-6) | Card padding (established pattern: `p-6` on MatchBreakdown card) |
| xl | 32px (p-8, gap-8) | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- AI explanation section top padding: `pt-4 mt-4` (16px each) with `border-t border-fog` separator — established in RESEARCH.md code example
- Dimension row spacing: `space-y-3` (12px) — established in MatchBreakdown.tsx
- Score circle bottom margin: `mb-2` (8px) — established in MatchBreakdown.tsx

## New UI Element: AI Explanation Section

This is the only net-new UI in Phase 4. It lives inside `MatchBreakdown.tsx` below the dimension rows.

### Layout

```
[MatchBreakdown card — existing]
  [score circle]
  [7 dimension rows with ProgressBars]
  ─────────────────────────────────   ← border-t border-fog
  Why this match                      ← 11px, --color-light, uppercase, tracking-wide
  {2-3 sentence explanation text}     ← 14px, --color-mid, leading-relaxed
```

### States

| State | Visual Treatment |
|-------|-----------------|
| Explanation present | Render section with border-t separator, label, and text |
| Explanation null (not yet generated) | Hide section entirely — no skeleton, no placeholder text |
| Explanation stale (score changed, new explanation pending) | Show previous explanation text — score numbers update immediately, text remains until replaced |
| Blurred overlay (visitor) | Explanation area is inside the blurred content div — not rendered separately |

### Interaction

- No interactivity on the explanation area — read-only text block
- No "regenerate" button — explanation updates are automatic and async
- No loading spinner — absence of explanation = section simply absent

## Interaction States

### MatchBreakdown with Explanation

| Trigger | Response |
|---------|----------|
| Page load, explanation exists | Explanation section renders below dimension rows |
| Page load, explanation null | Explanation section absent — no layout shift |
| Score updates (trigger fires) | Numbers in dimension rows update; explanation text remains (stale) |
| New explanation arrives (async) | Text replaces stale text — no animation required |

### Search Page Score Display

| State | Visual |
|-------|--------|
| Score present in match_scores | MatchCircle + score fraction rendered on SearchJobCard |
| Score absent (not yet computed, new job) | Score badge absent — SearchJobCard renders without score area |
| Score 0-39 | MatchCircle: red/low color (established MatchCircle pattern) |
| Score 40-69 | MatchCircle: orange/medium color |
| Score 70-100 | MatchCircle: green/high color |
