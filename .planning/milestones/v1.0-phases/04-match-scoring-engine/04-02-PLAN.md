**buildPrompt function** — accepts `totalScore: number` and `breakdown: Record<string, number>`, returns a string:
```
You are writing match feedback for a New Zealand farm job platform. Be honest, practical, and use plain farm-worker language. Never use corporate-speak or oversell.

Match score: ${totalScore}/100
Breakdown:
- Shed type: ${breakdown.shed_type}/25
- Location: ${breakdown.location}/20
- Accommodation: ${breakdown.accommodation}/20
- Skills: ${breakdown.skills}/20
- Salary: ${breakdown.salary}/10
- Visa: ${breakdown.visa}/5
- Couples bonus: ${breakdown.couples}/5

Write exactly 2-3 sentences explaining what drives this match score. Lead with the strongest factor. Be honest about gaps — if the score is low, say why clearly. Example style: "You've got rotary experience and they run a rotary shed — that's your strongest match. The salary sits at the top of your range. Location is the main gap — you're in Waikato and this role is Canterbury."

Only output the explanation sentences. No preamble, no labels, no bullet points.
```

**Handler function** (`Deno.serve`):
1. Handle OPTIONS preflight (return 'ok' with corsHeaders)
2. Parse `{ seeker_id, job_id }` from request JSON
3. Validate both fields present — return 400 if missing
4. Create Supabase service role client using `Deno.env.get('SUPABASE_URL')` and `Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')`
5. Load match score row: `.from('match_scores').select('breakdown, total_score').eq('seeker_id', seeker_id).eq('job_id', job_id).single()`
6. If not found, return 404 with `{ error: 'Score not found' }`
7. Create Anthropic client: `new Anthropic({ apiKey: Deno.env.get('ANTHROPIC_API_KEY') })`
8. Retry loop — 3 attempts with exponential backoff delays `[1000, 2000, 4000]`:
   - Call `anthropic.messages.create({ model: 'claude-sonnet-4-20250514', max_tokens: 150, messages: [{ role: 'user', content: buildPrompt(scoreRow.total_score, scoreRow.breakdown) }] })`
   - Extract text from `message.content[0]` if type is 'text'
   - On catch: increment attempt, sleep if more attempts remaining
   - Break on success (explanation is non-null)
9. If explanation obtained, upsert to match_scores: `.from('match_scores').update({ explanation }).eq('seeker_id', seeker_id).eq('job_id', job_id)`
10. Return 200 with `{ explanation }` (explanation may be null if all retries failed — this is graceful degradation, not an error)
11. Wrap entire handler in try/catch — on unexpected error, `console.error` and return 500 with `{ error: 'Internal server error' }`

All responses must include `{ ...corsHeaders, 'Content-Type': 'application/json' }` headers.

**Environment variables required:** `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ANTHROPIC_API_KEY`
