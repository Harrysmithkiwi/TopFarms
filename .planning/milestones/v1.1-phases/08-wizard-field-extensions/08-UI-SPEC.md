## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding, inline element gaps |
| sm | 8px | Gap between chips in a grid, compact label-input gap |
| md | 16px | Default field spacing, section padding, card padding |
| lg | 24px | Between form sections within a step, completion column gap |
| xl | 32px | Between major step sections |
| 2xl | 48px | Top/bottom padding on completion screens |
| 3xl | 64px | Not used in wizard context |

Exceptions:
- Chip buttons: `px-4 py-2` (16px × 8px) — horizontal 16px, vertical 8px, both standard set values
- InfoBox: `p-4` (16px) — established by InfoBox component
- Breadcrumb bar height (Phase 9): 44px — out of scope for Phase 8
- Toggle row container: `p-4` (16px) with fog border — established pattern from Step4Accommodation

Source: ChipSelector.tsx, InfoBox.tsx, Step4Accommodation.tsx — existing component patterns.

## Interaction Contracts

### Salary range inputs (EONB-07, PJOB-05)

- Two side-by-side inputs: "Min salary (NZD)" and "Max salary (NZD)"
- Validation: on blur, if min ≥ max, show inline error below max input: "Maximum must be greater than minimum"
- Market rate hint below the input pair: InfoBox variant="blue" — "Market rate for [role]: $55k–$75k NZD" — 13px text-ink
- Both fields optional (non-match-critical) per CONTEXT.md decisions

### Distance warning (EONB-02, PJOB-02)

- Trigger: distance-from-town select value is ">30km" or ">50km"
- Show: InfoBox variant="hay" below the distance select
- Copy: "Remote locations may receive fewer applicants. Consider highlighting accommodation and transport options."
- Disappears when distance value changes to ≤30km

### Availability / notice period nudge (SONB-04)

- Both fields are optional inputs
- Show encouragement nudge as InfoBox variant="blue" above or below the field group:
  "Adding your availability helps employers plan — listings with dates get 30% more views"
- Static; not conditional on field state

### LivePreviewSidebar (PJOB-07)

- Layout: CSS grid `grid-cols-[1fr_320px]` with 32px gap, container max-width unaffected
- Sidebar: `sticky top-6` — requires parent NOT to have `overflow: hidden` (documented in component)
- Visibility: hidden below 1024px (`hidden lg:block`) — mobile gets full-width form
- Mini card updates in real-time from react-hook-form `watch()` values: title, farm name, location, salary range, tags
- Completeness meter: percentage = (completed fields / total trackable fields) × 100, rounded to nearest 5%
  - Trackable fields for completeness: title, region, shed type, salary range, accommodation, description (6 fields)
  - Weights: all equal (each = ~17%)

### Boolean → ChipSelector migration

- DB migration converts boolean columns to `text[]` columns — no user-facing notice
- `booleanColumnsToChipArray()` utility maps existing boolean `true` values to corresponding string array entries before passing as `defaultValues` to ChipSelector
- Users re-opening wizards see their prior selections pre-populated as selected chips

### Annual/monthly billing toggle (EONB-08)

- Toggle row: same pattern as Step4Accommodation Toggle row — `bg-mist border-fog rounded-[10px] p-4`
- Left side: "Billing period" label (13px semibold ink) + subtext "Save 20% with annual billing" (12px mid)
- Right side: Toggle component with active=annual, inactive=monthly
- When annual selected: badge "Save 20%" in hay background, soil text, 11px, border-radius 4px, displayed inline next to the label

### Couples toggle sub-select (EONB-04)

- Main toggle: "Couples welcome?" — Toggle row pattern
- When enabled: reveal partner role sub-select (Select component) — "Partner's role" — slides in below with same `pl-1` indent pattern as Step4Accommodation conditional fields

### Wizard nav buttons

- All steps: `flex justify-between pt-2` footer — Back (ghost) left, Continue/final CTA (primary) right
- First step: Continue only, `ml-auto`
- Final step per wizard: primary button triggers save + navigate to completion screen
  - Employer onboarding final step: "Complete Profile"
  - Seeker onboarding final step: "Start Finding Jobs"
  - Post job wizard final step: "Post Job"
- Disabled state: `opacity-50 cursor-not-allowed` — built into Button component
