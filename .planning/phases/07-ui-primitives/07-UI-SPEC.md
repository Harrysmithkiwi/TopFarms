## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip icon-to-label gap, inline padding |
| sm | 8px | Chip internal padding (horizontal), button gap |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding within composed components |
| xl | 32px | Layout gaps between component sub-sections |
| 2xl | 48px | Major section breaks (SearchHero internal) |
| 3xl | 64px | Page-level spacing (not used within primitives) |

Exceptions:
- Pagination buttons: 34×34px fixed size (SPEC-exact, from PRIM-07 — not a spacing token, it is a component dimension)
- Breadcrumb bar: 44px height (SPEC-exact, from PRIM-03 / JDET-01)
- LivePreviewSidebar: 320px fixed width (SPEC-exact, from PRIM-09)
- Touch targets: minimum 44px per accessibility standard on interactive elements (StarRating, Pagination buttons — use padding to reach 44px tap area even if visual size is smaller)

Source: REQUIREMENTS.md PRIM-03, PRIM-07, PRIM-09; CONTEXT.md LivePreviewSidebar decision.

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | DM Sans (`font-body`) | 14px | 400 (regular) | 1.5 | Chip labels, stat values, banner body copy, breadcrumb items, pagination numbers, timeline entry text |
| Label | DM Sans (`font-body`) | 13px | 400 (regular) | 1.4 | Breadcrumb inactive items, stat labels below values, completeness meter percentage, AI tip text |
| Subheading | DM Sans (`font-body`) | 16px | 600 (semibold) | 1.4 | StatusBanner title copy, SearchHero input placeholder, LivePreviewSidebar section headings |
| Heading | Fraunces (`font-display`) | 28px | 300 (light) | 1.2 | SearchHero headline only |

Notes:
- Exactly 4 sizes declared: 13px, 14px, 16px, 28px
- Exactly 2 weights in active use across body roles: 400 (regular) and 600 (semibold). Fraunces 300 is used exclusively for the SearchHero headline.
- All body/label text uses `font-body` (DM Sans). Display heading uses `font-display` (Fraunces).
- Breadcrumb SPEC notation "11px, light" refers to `text-[11px] text-light` (the `light` color token, not font-light weight) — use 13px body for readability minimum; the color token `--color-light` (#9e8e78) is what makes inactive crumbs appear lighter.

Source: CONTEXT.md, RESEARCH.md code examples (breadcrumb spec note), REQUIREMENTS.md PRIM-07 (font-body text-[13px]).

## Color

| Dominant (60%) | `cream` | #f7f2e8 | Page/component background, chip default bg, sidebar bg |

Semantic colors in use for this phase:

- `meadow` (#7aaf3f): Timeline dots background, ProgressBar gradient terminus
- `purple-lt` (#f5eef8): AI tip box inside LivePreviewSidebar

## Component Visual Contracts

### PRIM-01: ChipSelector

- Container grid: controlled by `columns` prop → `flex flex-col gap-2` (1), `grid grid-cols-2 gap-2` (2), `grid grid-cols-3 gap-2` (3), `flex flex-wrap gap-2` (inline)
- Chip default state: `border-[1.5px] border-fog bg-white text-ink rounded-[8px] px-3 py-2`
- Chip hover state: `border-fern` (not moss — lighter green for hover affordance)
- Chip selected state: `border-moss bg-moss/7 text-moss` + checkmark icon (lucide `Check`, 14px, moss color)
- Checkmark position: absolute top-right corner of chip, or inline after label depending on chip width
- Optional icon: 20px, rendered left of label, color inherits from chip state (ink default, moss selected)
- Transition: `transition-all duration-150` on each chip
- Single-select: clicking a chip calls `onChange([chip.value])` — replaces entire array
- Multi-select: clicking a chip toggles presence in array

Source: CONTEXT.md ChipSelector decisions; RESEARCH.md ChipSelector code example; REQUIREMENTS.md PRIM-01.

### PRIM-02: StatusBanner

- Container: full-width, `rounded-[12px] border-[1.5px] p-4` layout
- Shortlisted: `bg-hay-lt border-hay` — title "Great news — you've been shortlisted!" — no action buttons (informational only)
- Interview: `bg-green-lt border-green` — title "You've got an interview invitation!" — CTA buttons: Accept (moss primary), Decline (ghost)
- Offer: `bg-green-lt border-green` — title "Congratulations — you've received an offer!" — CTA button: Review Offer (moss primary)
- Declined: `bg-red-lt/60 border-red-lt` — title "Unfortunately, this application wasn't successful." — no CTAs
- Title: 16px semibold, color inherits from variant (dark text for hay/green, `text-red` for declined)
- Body copy below title: 14px regular, `text-mid`
- CTA buttons inside banner: passed via `actions` prop (ReactNode) — not hard-coded in component

Source: CONTEXT.md StatusBanner decisions (exact copy strings, opacity requirement); REQUIREMENTS.md PRIM-02, MAPP-01.

### PRIM-03: Breadcrumb

- Container: `h-[44px] bg-white border-b border-fog flex items-center px-4 gap-2` (44px height per SPEC)
- Items: `text-[13px] font-body`
- Inactive item: `text-light hover:text-ink hover:underline` (clickable link)
- Active/current item: `text-ink font-medium` (not a link — no `href`)
- Separator: lucide `ChevronRight`, 14px, `text-light` — renders between each item pair
- Save/Share buttons: positioned absolute right side of bar, `text-[13px] text-mid hover:text-ink` with lucide `Bookmark` (16px) and `Share2` (16px) icons
- Maximum 4 items before truncation with `...` separator in middle position

Source: CONTEXT.md (Claude's Discretion for separator/active treatment); REQUIREMENTS.md PRIM-03, JDET-01; RESEARCH.md Breadcrumb interface example.

### PRIM-04: StatsStrip

- Container: CSS grid, `grid grid-cols-4 gap-0 border border-fog rounded-[12px] bg-white overflow-hidden`
- Each stat cell: `flex flex-col items-center justify-center py-4 px-2 border-r border-fog last:border-r-0`
- Stat value: `text-[20px] font-semibold text-ink font-body`
- Stat label: `text-[12px] text-light font-body mt-1`
- Responsive: at ≤860px → `grid-cols-2` with left-right pairs (Applications+Views, Salary+Posted)
- Supports 3 or 4 columns — controlled by `columns` prop (default 4)

Source: REQUIREMENTS.md PRIM-04, JDET-02; RESEARCH.md open question #1 (2-col breakpoint resolved to left-right pairs).

### PRIM-05: Timeline

- Container: `<ol className="relative ml-3 space-y-0">`
- Each item: `<li className="relative pl-6 pb-6 last:pb-0">`
- Dot: `absolute left-0 top-1 w-2.5 h-2.5 rounded-full bg-meadow` (meadow green, 10px circle)
- Connecting line: `absolute left-[4px] top-3.5 bottom-0 w-[1.5px] bg-fog` (fog color, hidden on last item via `last:hidden`)
- Entry title: `text-[14px] font-semibold text-ink font-body`
- Entry date/meta: `text-[13px] text-light font-body mt-0.5`
- Entry description: `text-[14px] text-mid font-body mt-1`
- No animation — static rendering only in Phase 7

Source: REQUIREMENTS.md PRIM-05, JDET-05; RESEARCH.md Timeline CSS-only code example.

### PRIM-06: StarRating

- Container: `flex items-center gap-0.5`
- Star icon: custom SVG path extracted from `TestimonialsSection.tsx` lines 33-47 (not lucide Star — ensures visual consistency with existing testimonials display)
- Star size: 16px default, `size` prop for override
- Filled state: SVG `fill="var(--color-hay)"` (#d4a843)
- Unfilled state: SVG `fill="var(--color-fog)"` (#eee8dc)
- Hover preview: hovering over star N shows stars 1–N as filled (hay color), stars N+1 onward remain fog
- Click: calls `onChange(rating: number)` where rating is 1–5
- No half-star support in Phase 7 — whole integers only
- Touch target: each star button minimum 44×44px tap area via padding

Source: REQUIREMENTS.md PRIM-06; CONTEXT.md (Claude's Discretion for hover); RESEARCH.md Pitfall 3, StarRating code example.

### PRIM-07: Pagination

- Container: `flex items-center gap-1`
- Page button (inactive): `w-[34px] h-[34px] flex items-center justify-center rounded-[6px] border-[1.5px] border-fog bg-white text-ink font-body text-[13px] hover:border-fern transition-colors`
- Page button (active): `border-moss bg-moss text-white`
- Prev/Next buttons: lucide `ChevronLeft` / `ChevronRight` (16px), same 34×34px container, disabled state: `opacity-40 cursor-not-allowed`
- Ellipsis: `text-[13px] text-light w-[34px] text-center` — rendered as static `...` text (not a button)
- Window algorithm: show first page, last page, and 3 pages centered around current page; `...` fills gaps
- Props: `currentPage: number`, `totalPages: number`, `onPageChange: (page: number) => void`

Source: REQUIREMENTS.md PRIM-07; RESEARCH.md Pagination code example + open question #2 (ellipsis resolved to sliding window).

### PRIM-08: SearchHero

- Container: `relative overflow-hidden` with inline style gradient (CSS vars must compose gradient via style prop — not pure Tailwind)
- Background: `linear-gradient(135deg, var(--color-soil) 0%, #1a3a10 100%)` + radial glow overlay `radial-gradient(ellipse at 30% 50%, rgba(74,124,47,0.25) 0%, transparent 60%)`
- Headline: `font-display text-[28px] font-light text-white leading-[1.2]` — centered above search bar
- Search bar container: `bg-white rounded-[12px] p-2 flex items-center gap-2 shadow-lg max-w-[680px] mx-auto`
- Text input: reuse `Input` component, placeholder "Search jobs, roles, farms…"
- Region select: reuse existing `Select` component, 8 NZ regions (Northland, Auckland, Waikato, Bay of Plenty, Hawke's Bay, Manawatu, Canterbury, Otago)
- Search CTA button: `bg-moss text-white rounded-[8px] px-4 py-2 text-[14px] font-semibold` — "Search"
- Quick-filter pills: below search bar, `flex flex-wrap gap-2 justify-center mt-4`
- Pill default: `bg-white/15 text-white border border-white/30 rounded-full px-3 py-1 text-[13px] cursor-pointer hover:bg-white/25 transition-colors`
- Pill active: `bg-white text-moss border-white`
- Pill labels: "Dairy", "Sheep & Beef", "Farm Manager", "Herd Manager", "Relief Milker" (default set, overridable via `pills` prop)
- Pill click: exclusive — calls `onPillClick?: (pill: string) => void` prop; Phase 9 wires navigation
- SearchHero search submit: calls `onSearch?: (query: string, region: string) => void` prop

Source: CONTEXT.md SearchHero decisions; REQUIREMENTS.md PRIM-08, SRCH-01; RESEARCH.md SearchHero gradient code example + open question #3.

### PRIM-09: LivePreviewSidebar

- Container: `sticky top-6 w-[320px] bg-white rounded-[14px] border border-fog overflow-hidden`
- Section header text: `text-[13px] font-semibold text-light uppercase tracking-wide px-4 pt-4 pb-2`
- Completeness meter section: percentage `text-[13px] font-body text-moss` + `ProgressBar` component (existing, moss→meadow gradient, `transition-all duration-300`)
- Mini card preview: simplified job card showing title (16px semibold), farm name (13px mid), location + salary (13px light) with 2-3 tag chips (fog bg, ink text, 11px)
- Match pool section: three static lines e.g. "47 seekers in region", "12 with shed experience", "8 actively looking" — `text-[14px] text-ink` + subtle note `text-[12px] text-light italic` "Estimates available soon"
- AI tip box: `bg-purple-lt rounded-[8px] p-3 mx-4 mb-4` — tip text `text-[13px] text-ink` — static placeholder "Tip: Listings with accommodation details get 40% more applications"
- Dividers between sections: `border-t border-fog`
- No async state — all values passed as props; Phase 11 wires live data

Source: CONTEXT.md LivePreviewSidebar decisions; REQUIREMENTS.md PRIM-09; RESEARCH.md Pattern 3 (composed component).
