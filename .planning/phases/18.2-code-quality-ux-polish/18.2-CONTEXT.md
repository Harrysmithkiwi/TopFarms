<decisions>
## Implementation Decisions

### SC-10: MarkFilledModal applicant picker UX

- **Identifier:** Show seeker name (from `seeker_profiles`) with email as fallback if name is blank. UUID fragment dropped entirely.
- **Additional data:** Show match score (from `match_scores`) alongside status. Both visible per row.
- **Layout:** Single line per radio row — `Name • Status badge • Score` (e.g. "John Smith • Shortlisted • 78pts").
- **Query change:** Extend the `.select('id, seeker_id, status')` at `MarkFilledModal.tsx:42` to join `seeker_profiles(first_name, last_name, email)` and `match_scores(score)`. If no seeker_profiles row, fall back to email.
- **UUID kept internally** for `selectedApplicantId` state — just not displayed to the employer.

### SC-11: UX-01 — Salary preset chips

- **Bands (fixed):** $50–60k, $60–70k, $70–80k, $80–90k, $90–100k, $100–110k, $110–120k, $120k+
- **Storage:** Lower bound of selected band stored in `min_salary` (e.g. selecting $70–80k → stores 70000). No schema change — existing integer column semantics preserved. Matching engine behaviour unchanged.
- **Selection mode:** Single-select — one band at a time. ChipSelector in single-value mode.
- **No free-text fallback:** Chips only. The 8 bands cover the full NZ ag salary range; no custom-value escape hatch.
- **Target file:** `src/pages/onboarding/steps/SeekerStep5LifeSituation.tsx` — replace the `<Input type="number">` for `min_salary` with a ChipSelector rendering the 8 bands.

### SC-2: Seeker nav — My Documents link

- **Route:** `/dashboard/seeker/documents`
- **Label:** "My Documents" (consistent with existing "My Applications" convention)
- **Both surfaces:** `Sidebar.tsx` (seekerItems array) + `Nav.tsx` (seekerLinks array) both get the new entry in the same position.
- **Icon (Sidebar.tsx):** Planner picks a suitable Lucide icon (e.g. `FolderOpen` or `FileStack` — matches existing icon style).
