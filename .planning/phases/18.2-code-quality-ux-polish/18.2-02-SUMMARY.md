## What Was Built

### Task 1 — SC-10: MarkFilledModal Applicant Picker

**Migration 030** (`supabase/migrations/030_rls_get_applicants_for_job_rpc.sql`):
- `get_applicants_for_job(p_job_id uuid)` SECURITY DEFINER RPC
- Returns `(id, seeker_id, status, display_name, match_score)`
- `display_name`: `COALESCE(first+last from seeker_contacts, email from auth.users, LEFT(application.id, 8))`
- Ownership check: caller must be employer who owns the job (via `user_roles` + `employer_profiles` JOIN)
- Written to disk; applied via Supabase Studio in plan 18.2-03 operator step

**MarkFilledModal.tsx** changes:
- `Applicant` interface extended with `display_name: string` and `match_score: number`
- Applicant list fetch replaced: `.from('applications').select(...)` → `.rpc('get_applicants_for_job', { p_job_id: jobId })`
- Row render replaced: `Applicant #{id.slice(0,8)}` → `{display_name} • <span className="capitalize">{status}</span> • {match_score}pts`

**Tests** (4 new, all passing):
- Renders "John Smith • Shortlisted • 78pts" for full-name applicant
- Renders "unknown@example.com • Applied • 0pts" for email-fallback
- Renders "ab12cd34 • Interview • 55pts" for UUID-fragment fallback
- "Applicant #" prefix never appears in any row

### Task 2 — SC-11: Salary Preset Chips

**SeekerStep5LifeSituation.tsx** changes:
- Added `SALARY_BAND_OPTIONS: ChipOption[]` with 8 entries (50000→$50-60k … 120000→$120k+)
- Imported `ChipOption` type from `@/components/ui/ChipSelector`
- Replaced `<Input type="number" {...register('min_salary')} />` with `Controller + ChipSelector (mode="single", columns="inline")`
- Value adapter: `field.value != null ? [String(field.value)] : []`
- onChange adapter: `vals[0] ? Number(vals[0]) : undefined`

**Tests** (3 new, all passing):
- All 8 salary band chip labels rendered; `input[type=number]` absent
- Clicking `$70–80k` → `onComplete({ min_salary: 70000 })` (number, not string)
- `defaultValues={{ min_salary: 80000 }}` → `$80–90k` chip has `border-brand` class

**1. [Rule 3 - Blocking] ResizeObserver polyfill in test setup**

- **Found during:** Task 2 RED phase
- **Issue:** `SeekerStep5LifeSituation` renders a Radix UI `Select` (notice_period_text field). The Radix `@radix-ui/react-use-size` hook calls `new ResizeObserver()` which jsdom does not implement. Tests threw `ReferenceError: ResizeObserver is not defined` blocking all 3 new tests.
- **Fix:** Added `global.ResizeObserver` stub (no-op observe/unobserve/disconnect) to `tests/setup.ts`. This is a global polyfill; pre-existing tests that render Radix components benefited from this fix.
- **Files modified:** `tests/setup.ts`
- **Commit:** 4ef6b49
