Create `supabase/migrations/031_rls_initplan_performance.sql`.

**The rewrite pattern is:**
- BEFORE: `USING (auth.uid() = user_id)` or `USING (auth.uid() = some_column)`
- AFTER: `USING ((select auth.uid()) = user_id)` or `USING ((select auth.uid()) = some_column)`

The `(select auth.uid())` subquery form prevents Postgres from re-evaluating `auth.uid()` as an initplan on every row. This is a performance-only change — no behaviour change.

**Critical rule:** Use `DROP POLICY IF EXISTS "name" ON table; CREATE POLICY "name" ON table ...` for every policy. Do NOT use ALTER POLICY — Postgres ALTER POLICY cannot change the USING expression.

**Migration structure:**

```sql

-- =============================================================
-- 031_rls_initplan_performance.sql
-- Phase 18.2 SC-13: Replace bare auth.uid() with (select auth.uid())
-- across all RLS policies to eliminate auth_rls_initplan advisor warnings.
-- Performance-only: no behaviour change to any policy.
-- Applied via Supabase Studio SQL Editor per CLAUDE.md §2.
-- Verify after apply: SELECT tablename, policyname, qual
--   FROM pg_policies WHERE schemaname='public'
--   AND qual LIKE '%auth.uid()%' AND qual NOT LIKE '%(select auth.uid())%';
-- Expected: 0 rows (all bare auth.uid() replaced).
-- =============================================================

-- user_roles
DROP POLICY IF EXISTS "user_roles: users read own" ON public.user_roles;
CREATE POLICY "user_roles: users read own"
  ON public.user_roles FOR SELECT
  USING ((select auth.uid()) = user_id);

-- employer_profiles (check 002_rls_policies.sql for exact policy names)

-- Repeat pattern for ALL seeker_profiles policies

-- 1. "seekers view own contact info" (seeker reads their own)
```
