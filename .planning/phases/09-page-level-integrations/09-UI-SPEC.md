## Interaction Contracts

### Job Search — Expandable Cards (SRCH-06)

- **Trigger:** Click anywhere on the card header row (not a link element)
- **Behaviour:** Accordion — one card open at a time; expanding a new card collapses the previous
- **Animation:** Smooth height transition, 200ms ease-out — do not use `display:none` toggle
- **Tab bar:** Details / My Match / Apply — rendered inside expanded area, below header
- **Active tab:** Bottom border indicator, moss colour, 2px thick
- **Tab visibility rules:**
  - My Match tab: visible only to logged-in seekers (hide entirely if unauthenticated)
  - Apply tab: hidden if seeker has already applied to this job
- **Quick-apply:** Optional cover note `<textarea>` (3 rows, max 500 chars) + "Apply Now" button — no modal; submit inline
- **Details tab truncation:** Job description preview truncated at 4 lines with `line-clamp-4`; "View Full Listing →" link at bottom

### Job Search — Active Filter Pills (SRCH-05)

- **Position:** Full-width row between SearchHero and results count line
- **Pill anatomy:** `bg-moss/10 border border-moss/30 text-moss text-[12px] rounded-full px-3 py-1` + `×` remove button
- **Remove behaviour:** Click × removes that filter; updates URL params; pill row disappears when no filters active
- **Overflow:** Wrap to next line if pills exceed row width — no horizontal scroll

### Job Search — Pagination (SRCH-08)

- **Position:** Centered below the results list
- **Sync:** Page number in URL param `?page=N`
- **Component:** Use existing `Pagination` component (34×34px buttons, fog border, active = moss bg)
- **Load More button:** Remove entirely; replace with `Pagination`

### Job Detail — Breadcrumb Bar (JDET-01)

- **Height:** 44px
- **Background:** white, `border-b border-fog`
- **Left:** Breadcrumb path using `Breadcrumb` primitive
- **Right:** Save (Bookmark icon, `text-mid hover:text-hay`) + Share (Share2 icon, `text-mid hover:text-moss`) buttons
- **Save state:** Bookmark icon fills `text-hay` when saved; toggles `saved_jobs` row

### Job Detail — Location Map Placeholder (JDET-06)

- **Height:** 160px
- **Background:** `bg-mist rounded-[12px]`
- **Content:** Centered `MapPin` icon (24px, `text-light`) + "Map coming soon" text (12px, `text-light`)
- **Distance badge:** Absolute-positioned bottom-right, `bg-white border border-fog rounded-full px-2 py-1 text-[12px] text-mid`

### Applicant Dashboard — Sidebar (ADSH-01)

- **Width:** 260px fixed; does not stack on tablet — DashboardLayout shell sidebar is hidden on this route
- **Farm header:** `bg-soil text-white`, farm name in Fraunces 16px, listing selector dropdown below
- **Listing selector:** Controlled dropdown; selecting a job updates `?job=xyz` URL param without full navigation
- **Quick stats:** 3-4 stat items (applied count, shortlisted count, hired count) in small grid

### Applicant Dashboard — 4-Tab Panels (ADSH-03)

- **Tab structure:** CV / Match Breakdown / Interview / Notes
- **CV tab:** Cover note display, seeker profile summary (reuse existing expanded content from `ApplicantPanel`)
- **Match Breakdown tab:** Full 7-dimension score bars using existing `MatchBreakdown` component
- **Interview tab:** Placeholder with "Interview scheduling coming in a future release" — do not build calendar
- **Notes tab:** Private employer notes textarea — stores in `application_notes` column
- **Tab active state:** Bottom border `border-b-2 border-moss`, active label `text-moss font-semibold`

### Applicant Dashboard — AI Candidate Summary (ADSH-04)

- **Trigger:** Generated when employer first expands the applicant panel (not background/batch)
- **Container:** `bg-purple-lt border border-purple/20 rounded-[12px] p-4`
- **Loading state:** 3 animated skeleton lines (`bg-purple/10 animate-pulse rounded h-3`) + "Analyzing candidate fit..." in 12px `text-purple`
- **Result:** 2-3 sentences in 13px `text-ink`. Purple label "AI Summary" (11px semibold, `text-purple`, uppercase) above
- **Cache:** Result stored in DB; subsequent views render instantly — no regenerate button

### Applicant Dashboard — Bulk Actions Bar (ADSH-05)

- **Visibility:** Appears when 1+ applicants are checkbox-selected
- **Position:** Sticky bottom of the applicant list area, `bg-white border-t border-fog`
- **Actions:** "Shortlist selected" (moss primary Button) + "Export" (ghost Button) — no "Send message" button
- **Selection count:** "N selected" label in 13px `text-mid` at left of bar

### My Applications — StatusBanner Position (MAPP-01)

- **Render condition:** Show at top of `ApplicationCard` when status is `shortlisted`, `interview`, `offered`, or `declined`
- **Margin:** 16px bottom margin between banner and card body
- **Interview variant:** Includes "Accept" (moss primary Button sm) + "Decline" (ghost Button sm) action buttons inside the banner `actions` prop slot
- **Declined variant:** 60% opacity wrapper (`bg-red-lt/60`) — established in `StatusBanner.tsx`

### My Applications — Farm Response Indicator (MAPP-02)

- **Position:** Below the applied date line inside `ApplicationCard`
- **Viewed:** "Viewed by employer 3 hours ago" — 12px `text-light`, Eye icon (14px, inline)
- **Not viewed:** "Not yet viewed" — 12px `text-light italic`
- **Data source:** `applications.viewed_at` column (set on employer first-open via upsert)

### Saved Jobs (Bookmark Toggle)

- **Surfaces:** Job search card header (Bookmark icon, 16px) + Job detail breadcrumb bar (Bookmark icon, 20px)
- **Unselected:** `text-mid hover:text-hay`
- **Selected:** `text-hay fill-hay` (filled icon)
- **Interaction:** Optimistic UI toggle; upsert/delete `saved_jobs` row in background
- **Error:** Toast "Could not save job — please try again" on failure; revert optimistic state
