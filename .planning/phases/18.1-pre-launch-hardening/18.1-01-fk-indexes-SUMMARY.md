## Decisions Made

- **Plain CREATE INDEX (not CONCURRENTLY):** Studio SQL Editor wraps multi-statement runs in an implicit transaction; `CONCURRENTLY` cannot execute inside a transaction. Pre-launch tables are sub-100-row counts — build time is millisecond-scale. Safe trade-off.
- **Studio apply deferred to Wave 3:** 025, 026, and 027 migrations batched for single operator context-switch in plan 18.1-06. Matches CLAUDE §2 preferred path (Studio SQL Editor over `--read-only` flag-flip cycle).
- **SC-6 count drift acknowledged:** ROADMAP wording says "14× unindexed_foreign_keys" but live Supabase advisor (2026-05-07 capture) returned 15. One extra index (`saved_jobs_job_id_idx`) landed vs ROADMAP estimate. Reconciliation deferred to phase verifier per 18.1-RESEARCH §Pitfall 6.
- **it.each() at GREEN-flip:** Wave 0 used 16 explicit `it.todo('declares index on public.<table>(<col>)')` rows for readability at scaffold time. Wave 1 GREEN uses `it.each(EXPECTED)('declares index on public.%s(%s)', (table, col) => ...)` — same 15 named cases, cleaner at real-body scale.
