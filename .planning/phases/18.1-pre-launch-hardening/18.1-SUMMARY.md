## Key Deviations from Plan

### Deviation 1: Vault approach for WEBHOOK_SECRET DB-level config (SC-3)

**Plan specified:** `ALTER DATABASE postgres SET app.webhook_secret = '<hex>'` via Studio SQL Editor.

**What happened:** Studio returned "permission denied" on the `ALTER DATABASE` DDL. Vault approach used instead: migration 029 (`vault.create_secret('WEBHOOK_SECRET', p_secret => '<hex>')`) applied via Studio. `handle_job_filled` reads `WEBHOOK_SECRET` from `vault.decrypted_secrets` directly. `current_setting('app.webhook_secret', true)` returns NULL (intentional — Vault is source of truth).

**MCP evidence:** `SELECT name, length(decrypted_secret) FROM vault.decrypted_secrets WHERE name='WEBHOOK_SECRET'` → `secret_len=64` ✓

**Impact:** Zero functional impact. Synthetic curl UAT 6/6 confirms the full chain works. The Vault approach is arguably more secure (encrypted at rest via pgsodium). SC-3 fully closed.
