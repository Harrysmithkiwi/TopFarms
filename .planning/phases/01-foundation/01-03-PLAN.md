<task type="auto">
  <name>Task 1: Create useAuth hook and AuthLayout component</name>
  <files>src/hooks/useAuth.ts, src/components/layout/AuthLayout.tsx</files>
  <action>
    **useAuth.ts** — central auth hook following the research pattern:

    ```typescript
    // State: session (Session | null), role (UserRole | null), loading (boolean)
    // On mount: getSession() + onAuthStateChange listener
    // When session exists: load role from user_roles table
    // Functions: signUp, signIn, signOut, resetPassword, updatePassword
    ```

    Implement these functions:

    1. `signUpWithRole(email: string, password: string, role: 'employer' | 'seeker')`:
       - Call `supabase.auth.signUp({ email, password, options: { data: { role }, emailRedirectTo: '${window.location.origin}/auth/verify' } })`
       - The handle_new_user trigger in the database creates the user_roles row from raw_user_meta_data.role
       - Do NOT manually insert into user_roles from client — the DB trigger handles this atomically
       - Return the signup result

    2. `signIn(email: string, password: string)`:
       - Call `supabase.auth.signInWithPassword({ email, password })`
       - onAuthStateChange will fire and update session + role

    3. `signOut()`:
       - Call `supabase.auth.signOut()`
       - Clear session and role state

    4. `resetPassword(email: string)`:
       - Call `supabase.auth.resetPasswordForEmail(email, { redirectTo: '${window.location.origin}/auth/reset' })`

    5. `updatePassword(newPassword: string)`:
       - Call `supabase.auth.updateUser({ password: newPassword })`

    Session persistence (AUTH-04): Supabase client stores session in localStorage by default with `persistSession: true` (already configured in supabase.ts). The onAuthStateChange listener re-hydrates on load via getSession().

    **AuthLayout.tsx** — split-screen layout per user decision:
    - Two-column layout: left panel (hidden on mobile, visible md+) and right panel
    - Left panel: `bg-soil` background, farm imagery placeholder (use a div with gradient from soil to soil-deep, text overlay with TopFarms branding and value prop text in cream/hay colours)
    - Right panel: `bg-cream` background, centered form area with max-width ~440px
    - Props: `children` (the form content), optional `title` and `subtitle` for the right panel header
    - Mobile: left panel hidden, right panel full width
    - Include TopFarms logo text at top of right panel: "TopFarms" in Fraunces font with meadow accent
  </action>
