## 4. Phase 17 — Saved Search

**Status:** Pending (directory exists, empty)
**Requirements:** SRCH-13, SRCH-14, SRCH-15 (3 requirements; orphaned at milestone level until phase plans exist)
**Depends on:** Nothing (independent feature work)

### Goal

A logged-in seeker can save the current filter combination as a named saved search, reload it later, and delete searches they no longer need.

### Success criteria (from ROADMAP.md)

1. Logged-in seeker with active filters can click "Save search", enter a name, and persist filter combo to DB
2. Seeker can view list of saved searches; clicking restores all filter state (URL updates, results refresh)
3. Seeker can delete a saved search; removed from DB and list
4. Saved searches persist across sessions (logout/login preserves them)

### Notes

- Reordered from former Phase 15 because gap closure took priority (MAIL-02 silently broken, PRIV-02 launch blocker)
- ROADMAP.md note: **plan this phase right before execution**, not now — to absorb any architectural learnings from email pipeline work (notification patterns may inform saved-search alert delivery via MAIL-06)

### Estimated scale

Comparable to a small v1.0 phase: 1-2 plans, new table, RLS policies, sidebar UI on JobSearch, ~3-5 commits.
