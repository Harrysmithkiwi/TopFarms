## 4. Match Scoring Engine (PRD v3.0 §9)

Pre-computed scores stored in `match_scores`. Recalculated on job insert/update or seeker profile update. Powers instant ranked search results.

### 4.1 Scoring Dimensions (Total: 100 points + bonuses)

| Dimension | Weight | Logic |
|-----------|--------|-------|
| Shed type match | 25 pts | Exact = 100%. Rotary↔Herringbone = 40% (transferable). AMS = separate pool. Most important single dimension. |
| Location | 20 pts | Same region = 100%. Adjacent region = 60%. Open to relocate = 80% base. |
| Accommodation match | 20 pts | Seeker needs + job provides = 100%. Sub-scored: housing type, pets, couples, family, utilities. |
| Skills & qualifications | 20 pts | Required skills = full weight. Preferred = 60%. Bonus = 20%. DairyNZ level match included. `willing_to_learn` gives partial credit. |
| Salary alignment | 10 pts | Job salary_max >= seeker min_salary = 100%. Partial credit for close ranges. Never penalises job paying above minimum. |
| Visa / eligibility | 5 pts | NZ citizen/PR = 100%. Working holiday on sponsored role = 100%. Ineligible = 0 (hard filter, not just low score). |
| Couples bonus | variable | Seeker couples_seeking + job couples_welcome = +5pt bonus on top of base score. |
| Recency multiplier | 1.1x | Jobs posted within 7 days get freshness boost. |

### 4.2 Display
- Match circles: moss green (>=80%), orange (60-79%), red (<60%)
- Fraunces serif percentage number inside circle
- Bar breakdown showing per-dimension scores on job detail page (logged-in users only)
- Claude API generates 2-3 sentence plain-language explanation from breakdown jsonb

### 4.3 Triggers
- Job insert/update → recalculate match_scores for matching-sector seekers
- Seeker profile update → recalculate match_scores for matching-sector jobs
- Nightly Edge Function → full recalculation batch for data integrity
- Performance target: search results with pre-computed scores return in <1.5 seconds
